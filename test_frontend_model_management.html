验证RAG系统前端模型管理和参数配置功能

📋 功能概览

本次更新在前端设置页面的"模型配置"部分添加了完整的模型管理功能，包括：

  • ✅ 统一的模型平台配置（API密钥、基础URL等）
  • ✅ 嵌入模型管理（添加、切换、测试）
  • ✅ 重排序模型管理（添加、切换、测试）
  • ✅ 模型状态监控和健康检查
  • ✅ 响应式界面设计
  • ⏳ 后端API集成（需要创建相应的API端点）

🎨 界面设计特点

  • 统一配置： 将相同平台的模型配置集中管理，避免重复配置
  • 直观操作： 提供下拉选择、刷新按钮、测试功能等便捷操作
  • 状态显示： 实时显示模型状态（🟢健康、🔴错误、🟡加载中）
  • 响应式设计： 适配不同屏幕尺寸，移动端友好

⚡ JavaScript功能

ModelManager类

主要功能：
  • 模型配置加载和刷新
  • 活跃模型切换
  • 新模型添加
  • 模型测试和健康检查
  • 状态监控和指标显示

全局函数

// 刷新模型列表 refreshEmbeddingModels() refreshRerankingModels() // 添加新模型 addEmbeddingModel() addRerankingModel() // 测试模型 testEmbeddingModel() testRerankingModel() // 状态监控 showModelStatus() showModelMetrics() performHealthCheck()

🔌 需要的API端点

前端功能需要以下后端API支持：

GET /api/model-manager/status
获取模型状态和配置信息
GET /api/model-manager/metrics
获取模型性能指标
POST /api/model-manager/switch-active
切换活跃模型
POST /api/model-manager/add-model
添加新模型配置
POST /api/model-manager/test-model
测试指定模型
POST /api/model-manager/health-check
执行健康检查
